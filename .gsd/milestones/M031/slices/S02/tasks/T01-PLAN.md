# T01: Parse parenthesized imports and add e2e tests for both features

**Slice:** S02 — Trailing Commas & Multiline Imports
**Milestone:** M031

## Description

The parser's `parse_from_import_decl` currently rejects `(` after `import` — it expects `IDENT` and emits `"expected import name"` when it sees `L_PAREN`. This blocks R018 (multiline parenthesized imports). R019 (trailing commas in fn call args) already works in the parser but has zero e2e test coverage. This task fixes the parser and adds tests for both features.

## Steps

1. **Modify `parse_from_import_decl` in `compiler/mesh-parser/src/parser/items.rs`** (line ~178). After consuming `import` keyword:
   - Check `p.at(SyntaxKind::L_PAREN)`. If true, `p.advance()` — this bumps `paren_depth`, making newlines insignificant automatically via the parser's `should_skip` logic.
   - Track whether parens were opened with a local `bool has_parens`.
   - The existing name-parsing loop stays the same.
   - Change the trailing-comma break condition from `if p.at(SyntaxKind::NEWLINE) || p.at(SyntaxKind::EOF) { break; }` to also include `|| p.at(SyntaxKind::R_PAREN)`. This is safe for both forms — `R_PAREN` never appears mid-import-list in the non-paren form.
   - After the name loop, if `has_parens`: `p.expect(SyntaxKind::R_PAREN)`.

2. **Add parser snapshot tests in `compiler/mesh-parser/tests/parser_tests.rs`:**
   - `from_import_paren` → `"from Math import (sqrt, pow)"`
   - `from_import_paren_trailing_comma` → `"from Math import (sqrt, pow,)"`
   - `from_import_paren_multiline` → `"from Math import (\n  sqrt,\n  pow\n)"`

3. **Run `cargo test -p mesh-parser --lib` and update snapshots** with `cargo insta review` or `UPDATE_SNAPSHOTS=1`.

4. **Add e2e tests in `compiler/meshc/tests/e2e.rs`** using `compile_multifile_and_run`. Use a helper module (`math.mpl` with `pub fn add(a :: Int, b :: Int) -> Int do a + b end` and `pub fn mul(a :: Int, b :: Int) -> Int do a * b end`). Tests:
   - `e2e_multiline_import_paren_basic` — `from Math import (add, mul)` on one line
   - `e2e_multiline_import_paren_multiline` — names on separate lines inside parens
   - `e2e_multiline_import_paren_trailing_comma` — `from Math import (\n  add,\n  mul,\n)`
   - `e2e_trailing_comma_call_single_line` — `add(1, 2,)` compiles and runs correctly
   - `e2e_trailing_comma_call_multiline` — multiline call with trailing comma

5. **Run full verification:** `cargo test -p mesh-parser --lib && cargo test -p meshc --test e2e && cargo run -p meshc -- build reference-backend && cargo run -p meshc -- build mesher`
