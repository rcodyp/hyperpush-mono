# M049/S03 Research — Generated `/examples` from scaffold output

## Summary

- **R116 is the real owner here.** The repo still has no `examples/` directory and no repo references to `/examples`, so S03 can stay narrow: create `/examples/todo-postgres` and `/examples/todo-sqlite`, prove they match fresh scaffold output exactly, and leave public-surface rewiring to S04.
- The dual-db scaffold is already stable enough to snapshot. `compiler/mesh-pkg/src/scaffold.rs` dispatches through `scaffold_todo_api_project_with_db(...)` (`999-1006`) into `scaffold_postgres_todo_api_project(...)` (`1010`) and `scaffold_sqlite_todo_api_project(...)` (`1170`).
- The existing M049 support harnesses already generate clean project trees via the **public CLI** with arbitrary project names:
  - `compiler/meshc/tests/support/m049_todo_postgres_scaffold.rs::init_postgres_todo_project(...)` (`133`)
  - `compiler/meshc/tests/support/m049_todo_sqlite_scaffold.rs::init_sqlite_todo_project(...)` (`116`)
- The scaffold output is **deterministic**. I compared retained `generated-project/` trees across different S01/S02 scenarios and found no `left_only`, `right_only`, or differing files for the same database mode. Exact recursive diffing is viable.
- Current tests prove scaffold shape and runtime behavior, but **nothing yet proves** that a committed example directory is byte-for-byte what `meshc init` emits today. S03 needs that stronger parity seam explicitly.
- One public-surface mismatch should be decided before snapshotting examples: the SQLite generated README template still opens with `This project was generated by \`meshc init --template todo-api\`` (`compiler/mesh-pkg/src/scaffold.rs:198`), while Postgres already uses the explicit `--db postgres` form. If `/examples/todo-sqlite` becomes evaluator-facing in S03, copying that generic string forward would reintroduce the ambiguity S02 removed from root docs and Mesh skills.

## Do Not Hand-Roll

- Do **not** seed `/examples` by copying a retained `.tmp/.../generated-project` snapshot and then renaming `todo-starter` by hand. The current init helpers already accept arbitrary project names, so generate `todo-postgres` and `todo-sqlite` directly.
- Do **not** prove parity with a few marker-string asserts. The output is already deterministic enough for exact recursive tree diffing.
- Do **not** use bare `meshc build examples/todo-*` in repo-owned proof rails. The examples will be tracked directories; builds need `--output` under `.tmp/...` to avoid churning repo files. Follow the existing `m046_route_free` / M049 support pattern.
- Do **not** add a repo-wide “generic `meshc init --template todo-api` is forbidden” grep. `scripts/fixtures/m047-s05-clustered-todo/README.md` still intentionally contains that historical string. Any new wording guard must be scoped to the public example READMEs or the live scaffold template, not the whole repo.

## Recommendation

1. **Fix the SQLite generated README top line before snapshotting examples.**
   - Update `compiler/mesh-pkg/src/scaffold.rs::todo_readme(...)` to use the explicit `--db sqlite` form.
   - Add a matching assertion in the scaffold/tooling tests so the public example surface does not drift back.

2. **Add one repo-owned example materializer/update command.**
   Best shape:
   - a small Node script under `scripts/` or `scripts/tests/`
   - generates `todo-sqlite` and `todo-postgres` into a temp root using the **public CLI**
   - accepts `MESHC_BIN` or `--meshc-bin` so tests can pass `CARGO_BIN_EXE_meshc`
   - supports both `write/update` and `check/diff` modes
   - replaces only `examples/todo-sqlite` and `examples/todo-postgres` after successful generation
   - stages into temp first because `create_project_dir(...)` fails if the target directory already exists

3. **Add a slice-owned parity/build/test rail in Rust.**
   Likely new target: `compiler/meshc/tests/e2e_m049_s03.rs`

   What it should prove:
   - fresh generated `todo-sqlite` == committed `examples/todo-sqlite`
   - fresh generated `todo-postgres` == committed `examples/todo-postgres`
   - `meshc test examples/todo-sqlite` reports `2 passed`
   - `meshc test examples/todo-postgres` reports `2 passed`
   - `meshc build examples/todo-sqlite --output <artifact/bin/...>` succeeds
   - `meshc build examples/todo-postgres --output <artifact/bin/...>` succeeds

4. **Keep S03 scoped.**
   Do not mix in:
   - root README/site rewiring to `/examples`
   - top-level proof-app retirement
   - additional runtime proof beyond build/test/parity
