**tests/e2e/regex_literal.mpl** (REGEX-01):

```
fn main() do
  let rx = ~r/\d+/
  let matched = Regex.match(rx, "hello123")
  if matched do
    println("digit found")
  else
    println("no digit")
  end
  let rx2 = ~r/[a-z]+/i
  let matched2 = Regex.match(rx2, "HELLO")
  if matched2 do
    println("case insensitive match")
  else
    println("no match")
  end
end
```
Expected output: `"digit found\ncase insensitive match\n"`

**tests/e2e/regex_compile.mpl** (REGEX-02):
Test Regex.compile with a valid pattern (Ok) and an invalid pattern (Err).
```
fn main() do
  let result1 = Regex.compile("\\d+")
  case result1 do
    Ok(rx) => println("compiled ok")
    Err(e) => println("error")
  end
  let result2 = Regex.compile("(unclosed")
  case result2 do
    Ok(rx) => println("should not succeed")
    Err(e) => println("got error")
  end
end
```
Expected output: `"compiled ok\ngot error\n"`

Note: In Mesh string literals, `\\d+` is a two-char escape sequence producing the literal string `\d+` which is the regex for digit.

**tests/e2e/regex_match.mpl** (REGEX-03):
Test Regex.match returning true and false.
```
fn main() do
  let rx = Regex.compile("^hello$")
  case rx do
    Ok(r) =>
      let m1 = Regex.match(r, "hello")
      if m1 do println("matched") else println("no match") end
      let m2 = Regex.match(r, "hello world")
      if m2 do println("matched") else println("no match") end
    Err(e) => println("compile failed")
  end
end
```
Expected output: `"matched\nno match\n"`

**tests/e2e/regex_captures.mpl** (REGEX-04):
Test Regex.captures returning Some with groups and None.
```
fn main() do
  let rx = Regex.compile("(\\w+) (\\w+)")
  case rx do
    Ok(r) =>
      let caps = Regex.captures(r, "hello world")
      case caps do
        Some(groups) =>
          let full = List.get(groups, 0)
          let g1 = List.get(groups, 1)
          let g2 = List.get(groups, 2)
          println(full)
          println(g1)
          println(g2)
        None => println("no captures")
      end
      let no_caps = Regex.captures(r, "123")
      case no_caps do
        Some(g) => println("unexpected match")
        None => println("no match")
      end
    Err(e) => println("compile failed")
  end
end
```
Expected output: `"hello world\nhello\nworld\nno match\n"`

Note: `List.get(groups, 0)` returns the full match (capture group 0), indices 1+ are the explicit groups. The `List.get` function returns an opaque value — it may return a raw pointer that must be treated as a String. Check how other list operations work in existing fixtures. If `List.get` returns `*mut u8` which needs to be printed as a string, it should work with `println` since Mesh type-checks that `groups` is `List<String>` and `List.get` returns `String`.
